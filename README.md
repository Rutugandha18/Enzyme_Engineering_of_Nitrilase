# Enzyme_Engineering_of_Nitrilase
Machine learning-guided mutation optimization framework for engineering nitrilase pH optimum properties
A two-stage computational pipeline for predicting and engineering the pH optimum of nitrilase enzymes using machine learning and evolutionary optimisation.

---

## Overview

This project addresses a core challenge in industrial enzyme engineering: nitrilases are highly useful biocatalysts, but their activity is tightly pH-dependent. Shifting the pH optimum — even by 0.5 units — can dramatically improve performance in a target bioprocess.

The pipeline has two stages:

**Stage 1 — ML pH Predictor** (`ml_pipeline_with_model_saving.py`)  
Trains regression models to predict pH optimum directly from protein sequence. Eight algorithms are benchmarked with Bootstrap augmentation to handle the small dataset size typical of curated enzyme databases. The best-performing model and its preprocessing artifacts (scaler, PCA, feature names) are saved to disk.

**Stage 2 — ACO-GA Sequence Optimiser** (`aco_ga_optimizer.ipynb`)  
Uses the saved model as a surrogate fitness function inside an Ant Colony Optimisation–enhanced Genetic Algorithm. The GA proposes amino acid substitutions; the surrogate predicts their pH; mutations that improve pH are reinforced. HotSpot Wizard mutability scores and coevolution data constrain which positions are mutated and how aggressively. A PSSM matrix biases substitutions toward evolutionarily tolerated amino acids.

## Stage 1 — ML pH Predictor

### Feature engineering

Each sequence is converted into a numeric feature vector before model training. The vector contains:

| Feature group | Description | Dimensionality |
|---|---|---|
| Biochemical | Length, MW, estimated pI, net charge at 6 pH values, GRAVY, aromaticity, aliphatic index, pKa stats | ~70 |
| Residue class counts & ratios | Acidic, basic, hydrophobic, polar, aromatic, aliphatic, tiny, small, charged, nonpolar | ~30 |
| Single-AA frequencies | Per-residue frequency for all 20 standard AAs | 20 |
| Dipeptides | All 400 dipeptide frequencies | 400 |
| Tripeptides | Observed tripeptide frequencies | variable |
| Tetrapeptides | Observed tetrapeptide frequencies | variable |
| Pentapeptides | Observed pentapeptide frequencies | variable |
| G-gap dipeptides | Gap-dipeptide frequencies for g = 2 … 10 | variable |

Features are standardised (StandardScaler) then compressed with PCA at 99% variance retention before model training.

### Data augmentation

Bootstrap resampling is applied with class-balanced sampling across three pH classes (acidic < 6.5, neutral 6.5–7.0, basic > 7.0) to address the small and imbalanced dataset typical of curated enzyme databases.

### Models benchmarked

Linear Regression · Ridge · Lasso · SVR (Linear, RBF, Polynomial) · Random Forest · Gradient Boosting

### Outputs

```

    model.pkl           trained sklearn estimator
    pca.pkl             fitted PCA transformer
    scaler.pkl          fitted StandardScaler
    feature_names.json  ordered feature column list
    metadata.json       R², RMSE, MAE, CV-R², dataset size
```

## Stage 2 — ACO-GA Sequence Optimiser

### How it works

The optimiser proposes amino acid substitutions to the wild-type sequence and uses the Stage 1 model to estimate the pH of each candidate — without any wet-lab measurement.

```
Wild-type sequence
      ↓
GA proposes mutant (e.g. V44Q, A63R, E122Q)
      ↓
extract_advanced_features()   ← same function used in training
      ↓
scaler → PCA → model.predict()
      ↓
Predicted pH  →  GA fitness
      ↓
ACO pheromone reinforces good substitutions
      ↓
Next generation ...
```

### Biological constraints

| Constraint | Source | Effect |
|---|---|---|
| Per-position mutability | HotSpot Wizard | Low-mutability positions mutated less aggressively |
| Coevolution pairs | HotSpot Wizard z-scores | Co-mutates partner residues during crossover |
| Substitution preference | PSSM (PSI-BLAST / HHblits) | Biases ACO heuristic toward evolutionarily tolerated AAs |
| Hamming distance cap | `MAX_HAMMING` parameter | Hard limit on number of simultaneous substitutions from WT |

### Key hyperparameters (set in Section 2 of the notebook)

```python
MODEL_NAME               = 'Gradient_Boosting'  # which saved model to load
GA_GOAL                  = 'maximize'            # 'maximize' or 'minimize' pH
POP_SIZE                 = 150
N_GENERATIONS            = 80
MAX_HAMMING              = 10                    # max substitutions from WT
PSSM_WEIGHT              = 0.5                   # 0 = ignore PSSM, 1 = PSSM only
HSW_MUTABILITY_THRESHOLD = 6                     # 0–9 scale
```

### Input files

| File | Format | Description |
|---|---|---|
| FASTA | `.fasta` | Single-record wild-type sequence |
| PSSM | `.xlsx` / `.csv` | Rows = 20 AAs (single-letter), columns = 1-indexed positions, values = substitution scores |
| HotSpot Wizard | `.csv` | Coevolution export with columns: `Position 1`, `Mutability 1`, `Position 2`, `Mutability 2`, plus z-score columns (`MI z-score`, `DCA z-score`, etc.) |

### Outputs

```
aco_ga_results.xlsx         6-sheet workbook:
    Summary                 run parameters + best result
    Top50_Sequences         top 50 unique mutants ranked by predicted pH
    Generation_History      per-generation convergence data
    Banked_Mutations        beneficial single substitutions observed
    Mutation_Frequency      most recurrent mutations across top-50
    HSW_Position_Hits       mutations at HotSpot Wizard coevolving positions

aco_ga_convergence.png      6-panel figure: pH convergence, annealing
                            schedules, memory system, Hamming diversity,
                            pH vs Hamming scatter, mutation frequency bar
```

## Usage

### Step 1 — Train and save models

Place your training data (`output_with_sequences.xlsx`) in the project folder, then:

```bash
python ml_pipeline_with_model_saving.py
```

This creates `saved_models/Bootstrap__<ModelName>/` directories for all 8 model types.

### Step 2 — Run the optimiser

Open `aco_ga_optimizer.ipynb` in Jupyter. Edit the paths and hyperparameters in **Section 2**, then run all cells. The predictor loads the saved model — no retraining.

```bash
jupyter notebook aco_ga_optimizer.ipynb
```

## Methods summary

> This pipeline implements a surrogate-model-guided evolutionary search for enzyme pH engineering. Sequence features (biochemical descriptors, k-mer frequencies up to pentapeptide, and g-gap dipeptides for g = 2–10) are extracted and compressed via PCA before regression. Bootstrap augmentation addresses class imbalance in the small training dataset. The ACO-GA optimiser integrates three sources of biological prior knowledge — PSSM evolutionary conservation, HotSpot Wizard mutability scores, and residue coevolution — to constrain the search to biologically tolerable sequence space while a Hamming distance cap prevents excessive divergence from the wild-type.
